# Firestore Schema - FinexyIA

## users (coleção)
- id (uid, string)
- name (string)
- email (string)
- photoURL (string)
- createdAt (timestamp)
- plan (string: 'Básico' | 'Premium')
- credits (number)
- notifications (map)
  - spendingAlerts (boolean)
  - monthlyReports (boolean)
  - dueAlerts (boolean)

## transactions (coleção)
- id (string, auto)
- userId (ref: users)
- amount (number)
- type (string: 'income' | 'expense')
- category (string)
- description (string)
- date (timestamp)
- createdAt (timestamp)

## goals (coleção)
- id (string, auto)
- userId (ref: users)
- name (string)
- targetAmount (number)
- currentAmount (number)
- monthlyTarget (number)
- history (array: {month: string, saved: number})
- createdAt (timestamp)

## reminders (coleção)
- id (string, auto)
- userId (ref: users)
- name (string)
- amount (number)
- dueDate (timestamp)
- createdAt (timestamp)

## cards (coleção)
- id (string, auto)
- userId (ref: users)
- brand (string)
- last4 (string)
- expMonth (number)
- expYear (number)
- createdAt (timestamp)

## rewards (coleção)
- id (string, auto)
- userId (ref: users)
- type (string)
- value (number)
- createdAt (timestamp)

## relationships
- Cada documento de transactions, goals, reminders, cards, rewards tem um campo userId referenciando o usuário dono.
